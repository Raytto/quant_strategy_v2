考虑建立一个量化策略分析网站，
- 网站入口 https://pangruitao.com/qs
    - 需要修改 nginx 反向代理配置
网站需要支持 查看/管理 各个已经存在的策略（如AH溢价策略等等）
- 可以查看策略的回测收益曲线，并且和一些常用指数进行对比（沪深300 纳指 恒生）
- 可以查看一些重点指标，比如年化收益、最大回撤、夏普率 等等
- 可以查看基于该策略，当下应该持仓哪些股票，以及各自的仓位占比（用凯利进行计算）
进一步网站需要支持多策略组合（类似多因子组合），可以在界面上勾选，把不同策略的选股逻辑整合起来，整体去进行凯莉优化，给出年化收益、最大回撤、夏普率、各个标的仓位占比
网站要求
- 相关数据（如把策略给变量化等等）可以存在 sqlite
- 网站页面用尽量简单的架构实现，不要 react，最好是最直接的 fastapi html css 结构
- 为了让各个策略都标准化，可能需要对各个策略代码本身进行进一步约束和要求设计
